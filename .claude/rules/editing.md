# Docs editing rules

## Source of truth: `app/_src/`, NEVER `app/docs/<version>/`

- `app/_src/` is single-sourced; `app/docs/2.x.x/` is generated by `jekyll-generator-single-source` (`app/_plugins/generators/`) and overwrites direct edits.
- Exception: `app/docs/dev/crds`, `app/docs/dev/protos`, `app/docs/raw/` are synced from the Kuma repo by `.github/workflows/` update jobs — do not edit by hand.

## Version-gate divergent content with `{% if_version %}`

Operators: `eq:`, `gte:`, `lte:`, `gt:`, `lt:`. Combine with a space: `"gte:2.4.x lte:2.8.x"`.

```liquid
{% if_version gte:2.10.x %}New feature only in 2.10+.{% endif_version %}
{% if_version lte:2.9.x %}Legacy instructions for 2.9 and older.{% endif_version %}
```

Gate from the version the feature landed in — content leaks into older versions otherwise.

## Never hardcode version URLs

Use `/docs/{{ page.release }}/path`. Forbidden: `/docs/2.10.x/...`, `/docs/2.10.0/...`. Enforced by `tools/check-version-urls.sh`. Legitimate exception: add `<!-- no-version-lint -->` on the same line.

## Register new pages in nav YAML

New `app/_src/` pages do not appear in the sidebar until added to `app/_data/docs_nav_kuma_<version>.yml`. Update the latest active version; backport only if the feature exists in older versions.

```yaml
- title: PolicyName
  url: /policies/policyname/
```

## Vale false positives

Add the term to `.github/styles/Vocab/`. Do not add new `<!-- vale off -->` regions.
